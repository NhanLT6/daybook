Where should the AI Chat Panel live?

This is the most fundamental layout decision — everything else builds on it. Click an option to select it.

A

Dedicated "AI Assistant" page

New nav item
Daybook Home Tasks AI Assistant ✦ Settings
Chat
Extracted Logs

A brand-new page in the nav bar. Full-screen layout gives the chat maximum space — split between the conversation on the left and extracted logs on the right. After reviewing, user can confirm and save the logs.

Pros

  • Maximum space for chat UX
  • Clean separation from log entry
  • Easier to build & test in isolation

Cons

  • Leaves Home view to chat
  • Extra navigation step
  • Chat history separated from logs
B

Persistent right panel inside Home

Your original idea
Daybook Home Tasks Settings
Calendar
Log Form
Log List
✦ AI Chat

The AI chat panel is always visible as a 4th column on the Home page, alongside Calendar, Log Form, and Log List. The chat sits right next to the log form — after AI extracts logs, user can click to auto-populate the form or save directly.

Pros

  • One-screen workflow (chat → save)
  • Log form is adjacent for quick review
  • Closest to your original vision

Cons

  • Home gets crowded (4 columns)
  • Chat column must share screen space
  • May need to collapse on narrow screens
C

Sliding overlay drawer (any page)

Global + on demand
Daybook Home Tasks Settings ✦ AI
Calendar
Log Form
Log List
✦ AI Chat

An AI button in the app bar toggles a slide-in drawer that overlays any page. The drawer can be opened from Home, Tasks, or anywhere. It dims the background but doesn't displace the layout — existing columns keep their full width.

Pros

  • Available from any page
  • Doesn't compress existing layout
  • Standard app pattern (familiar UX)

Cons

  • Covers content while open
  • Can't see log list & chat simultaneously
  • Slightly more implementation work
